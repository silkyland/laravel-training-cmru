CONTROLLERS

Controller Best Practices

หลักการเขียน Controller ที่ดีและมีประสิทธิภาพ

Single Responsibility

Controller มีหน้าที่เดียว

ดี รับ request → return response
ไม่ดี มี business logic มากเกินไป

Request Validation

Validate ก่อน process

public function store(StoreUserRequest $request)
{
  // $request->validated() พร้อมใช้
}

Keep Methods Short

ไม่เกิน 20 บรรทัด

• แต่ละ method ทำงานเดียว
• แยก logic ออกไป Services
• ใช้ Service Classes

Thin Controllers

Logic อยู่ใน Models/Services

Logic ควรอยู่ใน:
• Models - data logic
• Services - business logic
• Repositories - query logic

Return Responses

ใช้ Response Helpers

return response()->json($data);
return redirect()->route('users');
return back()->with('success', 'OK');

Authorization

ตรวจสอบสิทธิ์

$this->authorize('update', $user);
// หรือใช้ Gates / Policies
// @can('update', $post) ... @endcan

Dependency Injection

Inject ผ่าน Constructor

public function __construct(
  UserService $userService
) {
  $this->userService = $userService;
}

Route Model Binding

Auto-resolve Models

public function show(User $user)
{
  return view('users.show', compact('user'));
}
// Auto: User::findOrFail($id)

AI Integration

ใช้ AI ช่วย generate

• Generate Controller methods
• Create validation rules
• Write test cases
• Suggest refactoring
