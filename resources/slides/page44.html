Skinny Controllers, Fat Models

Bad:

Controller ทำ Database query ซับซ้อน, Validation, Business Logic...

Good:

Controller รับ Request -> คุยกับ Model/Service -> Return Response

SRP (Single Responsibility)

อย่าใส่ทุกอย่างใน Controller เดียว แยก Logic ออกไปที่:

  • Form Requests: สำหรับ Validation
  • Services: สำหรับ Business Logic ซับซ้อน
  • Observers/Events: สำหรับ Side effects (เช่น ส่งเมลหลัง Save)
  • Scopes: สำหรับ Query ที่ใช้ซ้ำ
