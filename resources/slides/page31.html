CSRF Protection

ความปลอดภัยของฟอร์ม

Laravel ปกป้องจาก Cross-Site Request Forgery โดยอัตโนมัติ ทุก POST/PUT/DELETE ต้องมี Token

Required: ใช้ @csrf ในทุกฟอร์ม
<form method="POST" action="/profile">
  @csrf
  <input name="name" />
  <button>Save</button>
</form>

Method Spoofing

PUT, PATCH, DELETE

HTML Form รองรับแค่ GET/POST จึงต้องใช้ @method ช่วย

PUT (Update):

<form method="POST" action="/post/1">
  @csrf
  @method('PUT')
  ...
</form>

DELETE:

<form method="POST" action="/post/1">
  @csrf
  @method('DELETE')
  <button>Delete</button>
</form>

Form Elements

Input Types ที่ใช้บ่อย

<!-- Text Input -->
<input type="text" name="title"
  value="{{ old('title') }}" />

<!-- Textarea -->
<textarea name="body">
  {{ old('body') }}
</textarea>

<!-- Select -->
<select name="category">
  @foreach($cats as $cat)
    <option>{{ $cat }}</option>
  @endforeach
</select>

Old Input

เก็บค่าเดิมเมื่อ validation fail

// In Blade
<input value="{{ old('name') }}" />
// With default
<input value="{{ old('name', $user->name) }}" />

CSRF Flow

กระบวนการทำงาน

1. Server สร้าง Token

2. @csrf ใส่ hidden field

3. Form submit พร้อม Token

4. Middleware ตรวจสอบ Token
