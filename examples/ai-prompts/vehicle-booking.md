# ระบบจองรถมหาวิทยาลัย (Vehicle Booking)

> Prompt สำหรับสร้างระบบจองรถราชการมหาวิทยาลัย ด้วย Laravel

---

## Prompt 1: วางแผนโครงสร้างทั้งระบบ

```
สร้างระบบจองรถราชการสำหรับมหาวิทยาลัย ด้วย Laravel 11, MySQL, Blade + TailwindCSS

ข้อมูลที่ต้องจัดการ:
- รถ: ทะเบียน, ยี่ห้อ, รุ่น, ประเภท (รถเก๋ง/รถตู้/รถบัส), จำนวนที่นั่ง, รูปภาพ, สถานะ (พร้อมใช้/ซ่อมบำรุง)
- พนักงานขับรถ: ชื่อ, เบอร์โทร, ใบขับขี่, สถานะ (ว่าง/ไม่ว่าง)
- การจอง: ผู้จอง, หน่วยงาน, รถ, พนักงานขับรถ, วันที่ไป, วันที่กลับ, สถานที่ปลายทาง, วัตถุประสงค์, จำนวนผู้โดยสาร, สถานะ (รออนุมัติ/อนุมัติ/ไม่อนุมัติ/เสร็จสิ้น/ยกเลิก)
- บันทึกการเดินทาง: เลขไมล์ก่อนออก, เลขไมล์กลับ, ค่าน้ำมัน, ค่าทางด่วน, หมายเหตุ
- ผู้ใช้: บุคลากร (จองได้), หัวหน้าหน่วยงาน (อนุมัติขั้นต้น), ผู้ดูแลยานพาหนะ (จัดรถ+คนขับ, อนุมัติขั้นสุดท้าย)

ฟีเจอร์:
1. CRUD รถ + พนักงานขับรถ (Admin)
2. ฟอร์มจองรถ: เลือกวันที่, ปลายทาง, จำนวนคน → แนะนำรถที่ว่างและเหมาะสม
3. ระบบอนุมัติ 2 ขั้น: หัวหน้าหน่วยงาน → ผู้ดูแลยานพาหนะ
4. ผู้ดูแลจัดรถและพนักงานขับรถให้
5. บันทึกการเดินทางหลังเสร็จสิ้น (เลขไมล์, ค่าใช้จ่าย)
6. ปฏิทินแสดงตารางรถทุกคัน
7. ค้นหารถว่างตามวันที่ + ประเภท + จำนวนที่นั่ง
8. รายงาน: สถิติการใช้รถแต่ละคัน, ค่าใช้จ่ายรายเดือน, ระยะทางรวม
9. แจ้งเตือน Email ทุกขั้นตอนอนุมัติ
10. Export รายงานเป็น PDF/Excel

ให้สร้าง:
1. Migration ทุกตาราง พร้อม Foreign Keys และ Indexes
2. Models พร้อม Relationships, Scopes, Accessors
3. Controllers: VehicleController, DriverController, BookingController, TripLogController, ReportController
4. Form Requests: StoreBookingRequest (ตรวจรถว่าง + วันที่ถูกต้อง)
5. Blade Views: ฟอร์มจอง, ปฏิทินรถ, หน้าอนุมัติ, บันทึกการเดินทาง, รายงาน
6. Middleware: CheckRole (admin/head/staff)
7. Notification: BookingStatusChanged (Email)
8. Routes: web.php แบ่งตาม Role
9. Seeders: รถตัวอย่าง 10 คัน, พนักงานขับรถ 5 คน
```

## Prompt 2: ระบบอนุมัติ 2 ขั้น

```
สร้างระบบอนุมัติการจองรถแบบ 2 ขั้นใน Laravel:

ขั้นตอน:
1. บุคลากรส่งคำขอจอง → สถานะ "รออนุมัติหัวหน้า"
2. หัวหน้าหน่วยงานอนุมัติ → สถานะ "รอจัดรถ"
3. ผู้ดูแลยานพาหนะจัดรถ + คนขับ + อนุมัติ → สถานะ "อนุมัติ"
4. หากขั้นใดไม่อนุมัติ → สถานะ "ไม่อนุมัติ" พร้อมเหตุผล

ให้สร้าง:
- Migration: เพิ่ม columns สำหรับ approved_by_head, approved_by_admin, rejection_reason
- Model: methods approve(), reject(), assignVehicle()
- Controller: methods สำหรับแต่ละขั้นตอน
- Blade: หน้าอนุมัติแสดงรายละเอียดคำขอ + ปุ่มอนุมัติ/ไม่อนุมัติ
- Notification: แจ้งเตือน Email ทุกครั้งที่สถานะเปลี่ยน
- Policy: ตรวจสิทธิ์ว่าใครอนุมัติขั้นไหนได้
```

## Prompt 3: บันทึกการเดินทางและรายงานค่าใช้จ่าย

```
สร้างระบบบันทึกการเดินทางหลังใช้รถเสร็จ:

ฟอร์มบันทึก:
- เลขไมล์ก่อนออกเดินทาง, เลขไมล์กลับ (คำนวณระยะทางอัตโนมัติ)
- ค่าน้ำมัน, ค่าทางด่วน, ค่าใช้จ่ายอื่น
- หมายเหตุ / ปัญหาที่พบ
- Validation: เลขไมล์กลับต้องมากกว่าเลขไมล์ออก

รายงาน:
- สรุปค่าใช้จ่ายรายเดือน แยกตามรถ
- ระยะทางรวมแต่ละคัน
- กราฟแท่ง Chart.js เปรียบเทียบค่าใช้จ่ายแต่ละเดือน
- Export เป็น PDF (DomPDF) และ Excel (Maatwebsite)

ให้สร้าง:
- Migration: trip_logs table
- Model: TripLog พร้อม relationship กับ Booking, Vehicle
- Controller: TripLogController (store, report)
- Blade: ฟอร์มบันทึก, หน้ารายงาน + กราฟ
- Export: TripLogExport class
```
